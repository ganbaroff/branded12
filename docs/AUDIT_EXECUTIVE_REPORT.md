# Исполнительный отчёт аудита веб-платформы B2Brand

**Дата:** 2026-02-24  
**Роль:** Агентство по разработке передовых веб-платформ  
**Объект:** Маркетинговая платформа (B2B, AZ/RU/EN), без онлайн-оплаты.

---

## Что проверено

Проведена проверка по семи категориям:

- **A. Функциональность и сценарии** — все публичные страницы (/, /services, /pricing, /contact, /brief, /portfolio, /cases, /quiz, /blog, 404), формы (бриф, чат-виджет, квиз), ценовые блоки, API и админка (вкладки, CRUD, экспорт лидов, 401).
- **B. Кнопки и интерактив** — hover/focus у ссылок и кнопок, пустые ссылки, CTA, состояния кнопок в админке.
- **C. Анимации и UX** — scroll-reveal (IntersectionObserver), lazy-load и Suspense, смена языка, таймеры акции (evergreen).
- **D. Безопасность** — заголовки (CSP, X-Frame-Options и др.), защита admin (authMiddleware, on401), rate limit и Zod для лидов, отсутствие секретов в клиенте.
- **E. Адаптивность и доступность** — адаптив навигации и форм, базовая a11y (фокус, семантика, ErrorBoundary).
- **F. Производительность и стабильность** — кэш публичных API, обработка ошибок API (apiGet/toast), зависимости useEffect в админке.
- **G. Контент и копирайт** — динамический год в футере, плейсхолдеры (WhatsApp, соцсети, аналитика).

## Критические проблемы

1. **Страница поста блога отсутствует**  
   На странице `/blog` карточки постов имеют кнопку «Oxu» / «Read more», но она не ведёт на отдельный пост. В приложении нет маршрута `/blog/:slug` и страницы с полным текстом поста, хотя API `GET /api/blog/:slug` реализован. Пользователь не может прочитать статью.

---

## Топ-5 приоритетных рекомендаций

1. **Добавить маршрут и страницу поста блога** — `Route path="/blog/:slug"`, страница запрашивает `GET /api/blog/:slug` и выводит контент; в карточке на Blog заменить кнопку на `Link to={/blog/${post.slug}}`.
2. **Блог: единый API-клиент и обработка ошибок** — в `Blog.tsx` использовать `apiGet` и при ошибке показывать toast (как на Services/Pricing), чтобы при падении API пользователь видел сообщение, а не пустой список.
3. **ErrorBoundary: локализация** — текст «Something went wrong» и «Reload page» выводить на текущем языке (AZ/RU/EN), используя контекст языка или сохранённый язык.
4. **Footer: явные стили focus** — у ссылок в футере добавить `focus:outline-none focus:ring-2 focus:ring-[#FF4D00]` (или аналог) для видимого фокуса с клавиатуры.
5. **CSP (желательно)** — по возможности уйти от `'unsafe-inline'` и `'unsafe-eval'` в script-src (nonce/hash для Vite), чтобы усилить защиту от XSS.

## Итог

Платформа в целом готова к использованию: формы работают, админка защищена, оплата отключена, таймеры и год актуальны. Единственная критическая недоработка — отсутствие страницы просмотра поста блога и ссылок на неё. Остальные замечания носят характер улучшений (a11y, обработка ошибок блога, локализация ErrorBoundary, ужесточение CSP).
